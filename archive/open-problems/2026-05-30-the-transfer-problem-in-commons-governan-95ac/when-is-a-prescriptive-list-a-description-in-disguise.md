The piece's central diagnostic move - that an inductively derived list of features from successful cases can confirm a regularity at the descriptive level without licensing the design-level use most commonly made of it - is not specific to Ostrom. The same structure governs many other domains where practitioners reach for a checklist drawn from successful exemplars: the "high-performance work practices" literature in industrial organization, the "characteristics of effective schools" literature in education, the WHO's "building blocks" framework for health systems, the World Bank's governance indicators, much of the design-patterns literature in software engineering. Each of these was generated by comparing successful cases and identifying shared features; each is then deployed in transplant contexts (advise a struggling firm / school / ministry to install the features).

The College's diagnostic instrument, if it works for Ostrom, would seem to apply directly to these other cases. What does the situational / action-formation / aggregation table look like for the "high-performance work practices" list? For the WHO health-systems building blocks? The morphologist's instinct here is that these prescriptive lists are doing the same operation Ostrom's design principles are - confirming a description-level regularity and then being asked to bear design-level weight - and that the mechanism gap is structurally identical. The piece does not claim to have generalized beyond commons governance; the question is whether the diagnostic apparatus *would* generalize, and whether a single follow-up piece could productively run the table on a second domain.
